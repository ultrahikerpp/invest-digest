### 各主題重點


#### Cloudflare 的企業定位爭議



- 作者認為，Cloudflare 難以被單一標籤定義，不同觀察者分別將其歸類為「資安公司」、「網路基礎設施」或「連線雲」，而 Cloudflare 官方採用「連線雲（Connectivity Cloud）」自稱。

- 作者認為，Cloudflare 的擴張邏輯極度嚴謹，表面看似大雜燴，實則遵循「先鋪設全球網路基礎設施，再在上面無極限疊加商業服務」的策略路徑。

- 作者認為，高盛最新 AI 基礎設施報告將 Cloudflare 列為首選，以及 Q1 財報後單日近 24% 的股價修正，是當前市場情緒分歧的兩個切入點，值得深入釐清。



#### 第一階段：WAF、DDoS 防護與 CDN 的免費策略



- 作者認為，Cloudflare 最初提供兩項核心服務：以 WAF 與 DDoS 防護扮演「網路門神」，攔截惡意流量；以 CDN 扮演「地方快遞員」，將靜態內容快取於鄰近機房以縮短載入時間。

- 作者認為，Cloudflare 的破局點在於將基礎服務免費開放給全球小型網站，這在當時被視為瘋狂，但實為深思熟慮的策略，因為這些免費用戶構成了全網的「早期預警雷達網」。

- 作者認為，全球 20% 網站流量通過 Cloudflare 基礎設施，其停機將導致數以百萬計網站癱瘓，足以說明其基礎設施的不可或缺性。



#### 數據飛輪效應



- 作者認為，Cloudflare 的核心護城河是「數據飛輪效應」：免費用戶貢獻攻擊數據 → 訓練防禦系統 → 防禦力增強吸引企業付費客戶 → 企業營收支撐更多機房擴建，形成自我強化的正循環。

- 作者認為，在資安產業中，數據本身就是最強的護城河，免費客戶並非成本負擔，而是整個商業模式不可或缺的原料來源。



#### 輕資產擴張模式



- 作者認為，Cloudflare 早期並非自建機房，而是與全球電信商（ISP）互惠合作，以免費幫電信商節省跨國頻寬成本為籌碼，換取在電信商機房內免費放置伺服器，以極低資本支出實現全球節點佈建。

- 作者認為，隨著業務轉向邊緣運算與 GPU 部署，Cloudflare 近年也開始租用或建置規格更高的大型 PoP，但早年輕資產打下的結構性優勢至今仍然成立。



#### 第二階段：邊緣運算與 Cloudflare Workers



- 作者認為，Workers 的核心革命在於將「即時動態運算」從遠端中央伺服器搬到離用戶最近的邊緣機房，將延遲從數百毫秒壓縮至個位數毫秒，解決了 CDN 無法處理的個人化即時計算問題。

- 作者認為，Workers 的成功建立在過去十年為 CDN 鋪設的 330 個城市節點之上，這些節點同時成為 330 個小型運算中心，是 AWS 短期內無法複製的物理優勢。

- 作者認為，Workers 採用「無伺服器（Serverless）」架構，讓開發者無需管理伺服器，程式碼自動跟隨用戶位置執行，是其相對於傳統雲端服務的差異化競爭點。



#### 邊緣開發者生態系



- 作者認為，Cloudflare 將邊緣運算平台構建為三層架構：底層為 Workers（通用運算引擎）、中間層為 R2、D1、KV 等資料服務、頂層為 Workers AI（邊緣 GPU 推論）。

- 作者認為，此三層架構使 Cloudflare 實質上成為一家完整的雲端運算公司，開發者可完全不依賴 AWS，直接在其平台上建構並運行應用程式。



#### 第三階段：零信任架構與企業資安



- 作者認為，COVID-19 疫情使傳統「城堡與護城河」式資安架構徹底失效，遠距工作催生了對零信任架構的龐大需求，成為 Cloudflare 商業版圖爆發性擴張的歷史催化劑。

- 作者認為，零信任的核心邏輯是「絕對不信任任何人，始終進行驗證」，無論員工身處何地，每次存取都需完成身份驗證、設備安全確認與權限控管。

- 作者認為，Cloudflare 進軍零信任的天然優勢在於，其遍布全球近 300 個城市的邊緣機房，可讓安全驗證的「檢查哨」就建在員工家門口，消除傳統雲端零信任的「髮夾彎效應」延遲問題。



#### 與 Zscaler 的競爭分析



- 作者認為，Zscaler 是零信任領域的先行者與現任市場領導者，根據 2024 年 Dell'Oro 的 SSE 市佔報告，Zscaler 以 34% 市佔遙遙領先，而 Cloudflare 甚至不在前六名之列。

- 作者認為，華爾街真正看重的不是當前排名，而是增長速度，而 Cloudflare 正明目張膽地搶佔 Zscaler 地盤，執行長 Matthew Prince 在法說會上直指其與「第一代零信任廠商」正面對決的勝率強勁。



#### 第四階段：R2 儲存與打破 AWS 資料綁架



- 作者認為，AWS 透過「資料傳出費（Egress Fees）」形成事實上的客戶綁架——存入免費，取出昂貴，導致開發者即使想採用 Cloudflare，也因資料鎖在 AWS 而無法完全遷移。

- 作者認為，R2 的「零傳出費」定位既來自 Cloudflare 與電信商 Peering 協議帶來的結構性頻寬成本優勢，也是有意識的戰略武器——以微利甚至虧損的儲存換取開發者生態入口。

- 作者認為，Cloudflare 並非全面挑戰 AWS，而是將競爭主戰場鎖定於「邊緣（Edge）」，訴求是讓開發者把需要速度與靠近用戶的前端任務交給 Cloudflare，重型核心運算仍留在 AWS。



#### AI 推論時代的邊緣佈局



- 作者認為，2023 年後 AI 熱潮的焦點在於「誰能訓練最大的模型」，但到 2026 年故事已轉變，AI 推論的物理學瓶頸成為新的核心議題，而 Cloudflare 的邊緣佈局正是針對這個瓶頸的戰略卡位。

- 作者認為，Workers AI 的定位是「讓 AI 推論離使用者最近」，在別人搶 GPU 訓練算力時，Cloudflare 搶的是推論服務的地理分佈優勢。

### 核心觀點



- 作者認為，理解 Cloudflare 的關鍵框架是「歷史累積的物理網路 → 疊加商業服務」的擴張邏輯：每一個新產品（Workers、零信任、R2、Workers AI）都不是憑空而來，而是建立在過去十年為 CDN 鋪設的全球節點之上，形成後人難以複製的結構性壁壘。

- 作者認為，「數據飛輪效應」是 Cloudflare 護城河的本質——免費用戶不是成本，而是數據原料；這套邏輯使其在資安領域的防禦能力持續自我強化，並以此為槓桿撬動企業付費市場。

- 作者認為，Cloudflare 的估值支撐邏輯在於，其過去十多年打下的全球邊緣基礎設施，正使其順理成章地成為「AI 時代的神經網路」，這是華爾街願意給予高估值並將其視為 Edge AI 最大受惠者的真正原因。

- 作者建議留意 **Cloudflare vs. Zscaler** 的企業市佔消長，特別是 Cloudflare 在 SSE 市場的排名與增速變化。

- 作者隱含建議投資人在按下交易鍵前，應清楚理解「自己買的是什麼、錢流向哪裡、風險的邊界在哪」，而非跟隨市場情緒單邊押注。
